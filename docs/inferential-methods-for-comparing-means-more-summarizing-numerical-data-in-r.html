Software Lesson 6 Inferential Methods for Comparing Means & More Summarizing Numerical Data in R

6.1 Goal of Software Lesson 6

The goal of this software lesson is to learn how to

  • obtain inferential results (e.g., CIs and p-values) for comparing two means: two-sample t-test and Mann-Whitney test/Wilcoxon rank-sum test.
  • summarize a numerical variable by group (descriptively and graphically), and
  • transform a numerical variable in a data set.


Let’s use the Lung Health Study (LHS) data set again.

Recall that the data set and a description of data can be found on Canvas, titled lhs.csv and lhsdoc.docx, respectively.

lhs<-read.csv(file = file.choose(), header = TRUE)

6.2 Two-sample t-test

Suppose that the LHS researchers wanted to investigate whether there was a difference in expired carbon monoxide (CO) level at baseline (CO0) between the male and female smokers (AGENDER). Because there are two separate, independent groups and you want to compare them on a numerical variable, this would be a situation for carrying out a two-sample t-test.

The following would be their research question:

Is there a difference in the population mean CO level at baseline between the male and female smokers?


To answer the research question, you would compute a confidence interval to estimate the difference in the population means between males and females and also carry out a hypothesis test for a difference in two population means. Both of these can be calculated by using the t.test() function (provided that the assumptions for the two-sample t-test are valid for the situation).

  • To carry out a hypothesis test for comparing two means (two independent groups), use the t.test() function, specifying the numerical variable first, followed by the ~ symbol (NOT the , symbol), and then the grouping variable, as well as all of the other potential arguments of the function, as deemed necessary (e.g., conf.level=, mu=, alternative=, var.equal=FALSE for unequal variance):
t.test(lhs$CO0~lhs$AGENDER, conf.level=0.95, 
       mu=0, alternative="two.sided", var.equal=FALSE)

The output for the two-sample t-test will look very similar to the output when you used t.test() to obtain inferential results for a single mean or mean difference.

Also notice that all of the usual arguments that you learned in a previous lesson also apply here (e.g., conf.level=, mu=, alternative=). The new argument, var.equal=, is, by default, set as var.equal=FALSE, or unequal variance assumption.


TWO-SAMPLE T-TEST

Purpose: To make an inference (CI or hypothesis testing) about the difference in population means between two groups.


For additional R resources on carrying out a hypothesis test for comparing two means, see:

  • The Independent Samples T-Test to Compare Two Means [BU-SPH].
  • Two-Sample T-Test in R: Independent Groups [MarinStatsLectures (youtube video)].
  • Unpaired Two-Samples T-Test in R [STHDA].

6.4 Mann-Whitney test (aka Wilcoxon Rank-Sum test)

Now suppose that the normality assumption was not met for carrying out a two-sample t-test. When this happens, you can use a nonparametric method that doesn’t assume normality: the Mann-Whitney test, also known as the Wilcoxon Rank-Sum test.

For the LHS study, suppose that the researchers were interested in whether there was a difference in salivary cotinine level at the 4th annual visit (COT4) between the male and female smokers (AGENDER). Also suppose that the conditions were not met to carry out a two-sample t-test. The Mann-Whitney test could be to used as an alternative approach for this situation.

Using the Mann-Whitney test, the following would be their research question:
Is there a difference in the median salivary cotinine levels at the 4th annual visit between the males and females?


Note: For this situation, the sample size is large enough that you don’t have to be concerned about the normality assumption (i.e., the CLT will hold because n is “large enough”). But let’s pretend that the normality assumption was an issue and another method was necessary.

  • To carry out Mann-Whitney test for comparing two medians (two independent groups), use the wilcox.test() function, specifying the numerical variable first, followed by the ~ symbol (NOT the , symbol), and then the grouping variable, as well as all of the other potential arguments of the function, as deemed necessary (e.g., conf.int=, conf.level=, mu=, alternative=):
wilcox.test(lhs$COT4~lhs$AGENDER, conf.int=TRUE, conf.level=0.95, alternative="two.sided", mu=0)

The output provides a test statistic and a p-value from the test results.


MANN-WHITNEY TEST

Purpose: To make an inference (hypothesis testing) about the difference in population medians between two groups. This is the nonparametric version to the two-sample t-test, if the normality condition wasn’t met.


For additional R resources on carrying out a Mann-Whitney test, see:

  • Mann-Whitney-Wilcoxon Test [R Tutorial].
  • Unpaired Two-Samples Wilcoxon Test in R [STHDA].

6.6 Summarizing numerical variables by group

In a previous lab, you learned how to create a graphical displays (e.g., boxplot, histogram) and calculate summary measures (e.g., mean, median, sd) for a single numerical variable. However, it may be of interest to compare groups on the numerical variable. This can be done with functions that summarize a variable by group or by multivariate plots (e.g., histograms by group, side-by-side boxplots).

The two LHS variables that will be the focus in this section are the age variable, age, and the intervention group variable, alphagroup.

6.6.1 Summary measures of numerical variable by group

There are two ways to calculate summary measures of a numerical variable by group. The first option is to use the tapply() function, which allows you to calculate single measures at a time, and the second option is to use the describeBy() function, which calculates multiple summary measures at once.

Option #1

  • To compute summary measures of a numerical variable by group, use the tapply() function, specifying the numerical variable in the first argument (e.g., lhs$age), followed by the grouping variable (e.g., lhs$alphagroup), and then adding the function of the summary measure that you want computed (e.g., median) in the last argument:
tapply(lhs$age, lhs$alphagroup, median)

Other summary measure functions that may be applied in the last argument of the tapply() function include mean,sd, IQR, max, and min.

R TIP

Recall that the summary measure functions above will return \(NA\) for any variable that has missing values. In those situations, you will need to include a na.rm=TRUE as the last argument in the function (e.g., tapply(object$num.var, object$grp.var, function, na.rm=TRUE)).

For example, the LHS variable for salivary cotinine at 4th annual visit ( COT4) has missing values. To compute the mean for this variable by intervention group, you would type the following code: tapply(lhs$COT4, lhs$alphagroup, mean, na.rm=TRUE).


Option #2

  • To compute summary measures of a numerical variable by group, use the describeBy() function from the {psych} package, specifying the numerical variable in the first argument (e.g., lhs$age), followed by the grouping variable (e.g., lhs$alphagroup) in the last argument [Note: You can also compute IQR and quantiles (e.g., Q1 and Q3) by adding additional (optional) arguments, IQR=TRUE and quant=c(0.25, 0.75)]:
###Access the package using the library() function
library(psych)

###Calculate multiple summary statistics by group using the describeBy() function
describeBy(lhs$age,lhs$alphagroup, IQR=TRUE, quant=c(.25,.75))

The describeBy() function is related to the describe() function you learned in a previous lab. It outputs the 12 summary statistics that describe() does, plus the three additional ones (IQR, Q1 (Q0.25), and Q3 (Q0.75)), but computes them by group.

6.6.2 Graphical displays of numerical variable by group

The two useful graphical displays that are used to compare the distributions of a numerical variable by group are the side-by-side boxplot and multiple histograms by group.

Side-by-side boxplot

  • To create a side-by-side boxplot of a numerical variable by group, use the boxplot() function, specifying the numerical variable first, followed by the ~ sign (NOT the , symbol), and then the grouping variable next, and other arguments if you wish to customize the bar plot (e.g., main= for the plot title):
boxplot(lhs$age~lhs$alphagroup, main="Boxplot of LHS Ages by \n Intervention Group")

Multiple histograms

  • To create histograms of a numerical variable by group, use the histogram() function from the {lattice} package, specifying the the ~ sign first, followed by the numerical variable and the | symbol, and then the grouping variable, and other arguments if you wish to customize the bar plot (e.g., xlab= for the x-axis label, main= for the plot title):
#############
# Run the install.packages() function only once per package
#############
install.packages("lattice")

###Access the package using the library() function
library(lattice)

###Create multiple histograms using the histogram() function
histogram(~lhs$age | lhs$alphagroup, 
          xlab="Age", main="Histograms of Age by \n Intervention Group")

6.8 Transforming a numerical variable

In a previous lab, you created a new variable in a data set of the difference between two variables for each observation. Recall that do this, the beginning of the code looked like dataset$newvariablename<-.

Now, you are going to learn how to transform an existing numerical variable into a new one, using the log-transformation.

For example, suppose you want to log-transform a skewed variable, such as the LHS variable of average number of cigarettes per day at 5th annual visit (AV5CIGS).

  • To log-transform a numerical variable in the data set, create a new variable in the data set (e.g., lhs$logAvgCigs.5), followed by the assignment operator (<-), and then use the log() function, specifying the variable that you want to take a log of (e.g., lhs$AV5CIGS):
lhs$logAvgCigs.5<-log(lhs$AV5CIGS)

Now, when the head() function is called, you will see that R attached a new column at the end of the data set (i.e., logAvgCigs.5). You can now apply any of the functions you have learned on this new numerical variable (e.g., summary(lhs$logAvgCigs.5), hist(lhs$logAvgCigs.5)).

R TIP

When naming new objects or variables in R, don’t use underscores (\(\_\)) or hyphens (\(-\)). Also, don’t start the names with a number. The preferred format is all lower case letters and words separated with dots (variable.name), but variableName is also acceptable, as are title case (VariableName).
