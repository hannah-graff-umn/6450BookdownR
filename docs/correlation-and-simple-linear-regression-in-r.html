Software Lesson 12 Correlation and Simple Linear Regression in R

12.1 Goal of Software Lesson 12

The goal of this software lesson is to learn how to

  • create a scatterplot,
  • obtain a correlation coefficient,
  • obtain regression coefficients (i.e., intercept and slope) and \(R^2\) for a simple linear regression model, and
  • subset the data.


Let’s use the Lung Health Study (LHS) data set again.

Recall that the data set and a description of data can be found on Canvas, titled lhs.csv and lhsdoc.docx, respectively.

lhs<-read.csv(file = file.choose(), header = TRUE)

12.2 Relationship between two numerical variables

Thus far, you have examined the relationship between a numerical variable and a categorical variable (e.g., two-sample t-test, ANOVA) and also between two categorical variables (e.g., Chi-square test, Fisher’s exact test, McNemar’s test). Now you are going to investigate the relationship between two numerical variables (e.g., scatterplot, correlation, simple linear regression).

In the LHS study, suppose the researchers wanted to examine the relationship between the number of cigarettes smoked per day at baseline (f10cigs) and FEV1/FVC\(\%\) at baseline (FEVFVC02). This will be the scenario used throughout this section of the lesson.

12.2.1 Scatterplot

The first step when examining the relationship between two numerical variables is to make a graphical display called a scatterplot.

  • To make a scatterplot, use the plot() function, specifying the x-variable first and then the y-variable second, separated by a , (e.g., plot(x, y)), and other arguments if you wish to customize the scatterplot (e.g., xlab= to specify the x-axis label, ylab= to specify the y-axis label, main= to specify the plot title):
plot(lhs$f10cigs, lhs$FEVFVC02, 
     xlab="Cigarettes per day", 
     ylab="FEV/FVC%", 
     main= "FEV/FVC% vs. Cigarettes at Baseline")

SCATTERPLOT

Purpose: To graphically display the relationship between two numerical variables.


For additional R resources on scatterplots, see:

  • <a href”https://www.statmethods.net/graphs/scatterplot.html“>Scatterplots [Quick-R].
  • Scatterplots - R Base Graphs [STHDA].
  • Easy scatterplot using ggplot2 [STHDA] [Note. The {ggplot2} package is a very useful package for creating elegant and aesthetically pleasing visualizations].


12.2.2 Correlation

If the relationship between the two numerical variables looks approximately linear, you can quantify the relationship by computing a correlation coefficient, r.

  • To compute the correlation coefficient, use the cor() function, specifying the x- and y-variables, separated by a ,, and the argument to delete cases that have a mission value in either of the two variables (use="complete.obs"):
cor(lhs$f10cigs, lhs$FEVFVC02, use="complete.obs")

The use="complete.obs" argument is required in the cor() function if there are missing values in either of the two variables. Without it, R would output NA for the correlation, which you know isn’t true. If there are not missing values in either of those two variables, then you can leave this argument out of the function.

CORRELATION

Purpose: To quantify the relationship (strength and direction) between two numerical variables.


For additional R resources on correlation, see:

  • Correlation Test Between Two Variables in R [STHDA].
  • Correlation [BU SPH].

12.3.1 Simple linear regression

Besides correlation, another way to describe the relationship between two numerical variables is through simple linear regression (SLR) if the relationship looks approximately linear. In SLR, you obtain regression coefficients (i.e., intercept and slope) from the SLR model to predict Y from X. For example, you can create a SLR model to predict FEV1/FVC\(\%\) at baseline (Y) from the number of cigarettes smoked per day at baseline (X).

  • To fit a simple linear regression model, first, use the lm() function, specifying the y-variable and the x-variable, separated by ~ (e.g., lm(y~x)) and saving the information under an object name (e.g., model1). Then use the summary() function on the saved lm object (model1) to obtain the regression coefficient values:
# fitting a SLR line
model1<-lm(lhs$FEVFVC02~lhs$f10cigs)

# obtaining results in SLR object, model1
summary(model1)

The main information of interest is in the table under the header Coefficients:.

The regression coefficient values are under the column Estimate, with the intercept denoted in the first row ((Intercept)) and the slope denoted in the second row. The information in this column is of most interest this week.

The standard errors for each of the regression coefficients are under the column Std. Error. This value is used to create confidence intervals and t-test statistics for the coefficients.

In the third column, t value represents the t-test statistics for testing the regression coefficients.

And lastly, the p-values for testing the regression coefficients are presented in the column Pr(>|t|). These last three pieces of information will be discussed next week.

Additional model statistics that may be of interest include \(R^2\), which is denoted next to Multiple R-squared, and \(\hat{\sigma}\), which is denoted next to Residual standard error. \(R^2\) is a measure of how well the model fits the data and the residual standard error, \(\hat{\sigma}\), is the standard deviation of the residual values.

12.3.2 Draw line of best fit

The saved lm object can also be used to draw a line of best fit in a scatterplot.

  • To add the line of best fit to a scatterplot, first, use the plot() function, specifying the x- and y-variables and other arguments if you wish to customize the scatterplot, and then use the abline() function on the saved lm object:
plot(lhs$f10cigs, lhs$FEVFVC02, 
     xlab="Cigarettes per day", 
     ylab="FEV/FVC%", 
     main= "FEV/FVC% vs. Cigarettes at Baseline")
abline(model1)

Purpose: To estimate regression coefficients to examine (and predict) the relationship between two numerical variables: a response variable, Y, and an explanatory variable, X.

12.5 Subsetting a data set

Sometimes when you are working with a large data set, you are only interested in a small portion of it. For example, suppose the LHS researchers were only interested in the females that were in the study.

There are two ways to subset the data in R. The first option is the most basic way by using square brackets after the data name, data[rows, columns]. The second option (and easiest to read) is to use the subset() function.

12.5.1 Option #1

  • To subset the data set, first, create a new object/data set in R (e.g., lhs.f1), followed by the assignment operator (<-), then assign the new data set to the old one (e.g., lhs), followed by which rows you are interested in keeping (e.g., lhs[lhs$AGENDER=="F", ]):
lhs.f1<-lhs[lhs$AGENDER=="F", ]

Here’s how you read the line of code:

“I want to create a new object/data set, called lhs.f1, (denoted by lhs.f1<-) … and this object is the lhs data set (lhs[ , ]), but only give me the rows for when AGENDER==”F” (denoted by the lhs$AGENDER=="F" before the comma in the square brackets), and give me all of the columns (denoted by the empty space after the comma in the square brackets).”

12.5.2 Option #2

  • To subset the data set, first, create a new object/data set in R (e.g., lhs.f2), followed by the assignment operator (<-), then use the subset() function, specifying the name of the data set first and then how you want to subset the data (e.g., subset(data, data$variable == value):
lhs.f2<-subset(lhs, lhs$AGENDER=="F")

Note: For either subsetting option, if you want to subset the data on a range of values for a numerical variable (e.g., age > 50) rather than on a single value (as presented above), then you can just supply that particular criteria in the code.
