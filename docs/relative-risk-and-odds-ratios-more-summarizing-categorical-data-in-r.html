Software Lesson 10 Relative Risk and Odds Ratios & More Summarizing Categorical Data in R

10.1 Goal of Software Lesson 10

The goal of this software lesson is to learn how to

  • summarize two categorical variables (descriptively and graphically), including 2x2 tables and plots,
  • calculate point and interval estimates for comparing two categorical variables (i.e., risk difference, relative risk, and odds ratio), and
  • modify labels of a categorical variable.


Let’s use the Lung Health Study (LHS) data set again.

Recall that the data set and a description of data can be found on Canvas, titled lhs.csv and lhsdoc.docx, respectively.

lhs<-read.csv(file = file.choose(), header = TRUE)

10.2 Comparing two categorical variables

Suppose that the LHS researchers were interested in estimating the relative risk of using nicotine gum at the 1st annual visit (AV1GUM), comparing females to males (AGENDER). This is a situation for using techniques that compare two groups on a categorical variable.

The following would be their research question:

What is an estimate of the population difference in risks/odds of using nicotine gum at 1 year after starting treatment between the sexes?


To answer the research question, you could compute and estimate the following summary measures: risk difference (RD), relative risk (RR), or odds ratio (OR), depending on the study design. However, before you compute these values, you should first examine the relationship between two categorical variables using 2x2 (or contingency) tables and stacked or side-by-side bar plots.

10.3 Summarizing two categorical variables

In a previous software lesson, you learned how to create a table and bar plot for a single categorical variable. However, it may be of interest to examine the relationship between two categorical variables. This can be done with 2x2 (or contingency) tables and by stacked or side-by-side bar plots.

10.3.1 2x2 tables

  • To create a 2x2 table of counts, use the table() function, specifying the two variables of interest:
table(lhs$AGENDER, lhs$AV1GUM)

The first variable in the argument (e.g., lhs$AGENDER) is placed in the rows and the second variable in the argument (e.g., lhs$AV1GUM) is placed in the columns.

If you want the 2x2 table to be proportions rather than counts, there are two ways to do this. The first option, prop.table(), allows you to create three different kinds of 2x2 tables, depending if you want proportions by total (cell value divided by total), proportions by row (cell value divided by row total), or proportions by columns (cell value divide by column total). The second option, CrossTable(), gives you all of that information in a single table.

Option #1: prop.table() for table of proportions

When creating a 2x2 table of proportions using the prop.table() function, you first need to save the 2x2 table of counts from the table() function into an object (e.g., mytable) and then use that object in the function for computing table proportions (prop.table()).

  • Step 1: To save the 2x2 table of counts into an object, first, name the object something meaningful (e.g., mytable), and then use the assignment operator (<-) to assign the 2x2 table of counts (e.g., table(lhs$AGENDER, lhs$AV1GUM)) to the object:
mytable<-table(lhs$AGENDER, lhs$AV1GUM)
  • Step 2: To create a 2x2 table of proportions, use the prop.table() function, specifying the saved object containing the 2x2 table of counts (e.g., mytable) and other arguments if you want specific proportions (e.g., margin = 1 for proportions by row, margin = 2 for proportions by column):
prop.table(mytable) # cell proportions, cell/total
prop.table(mytable, margin = 1) # row proportions, cell/row total
prop.table(mytable, 2) # column proportions, cell/column total

Row proportions and column proportions are helpful when you want to know the proportions given a particular subcategory. For example, based on the output from the row proportions code, you could answer the question of, “what proportion of female participants were using nicotine gum at 1 year after starting treatment?”

Option #2: CrossTable() for table of proportions

If you are looking for function that provides a multitude of information at once (counts, proportions), then use the CrossTable() function. The author that created this function modeled it after output provided by SAS for 2x2 tables (proc freq task in SAS).

  • To create a 2x2 table with multiple values in each cell, use the CrossTable() function from the {gmodels} package, specifying the two variables of interest:
#############
# Run the install.packages() function once per package
#############
install.packages("gmodels")

###Access the package using the library() function
library(gmodels)

###Calculate multiple summary statistics using the CrossTable() function
CrossTable(lhs$AGENDER, lhs$AV1GUM)

The values computed by this function include count, Chi-square contribution (ignore for now), proportion of the row, proportion of the column, and proportion of the table for each cell, respectively, for each cell in the table.

10.3.2 Graphical displays of a categorical variable by group

After creating a 2x2 table, it might be helpful to visually examine the data via a bar plot. Two types of bar plots include stacked bar plots and side-by-side bar plots. The function is the same as you learned in a previous software lesson, barplot() but now you use your 2x2 table object.

Stacked bar plot

  • To create a stacked bar plot of counts by group, use the barplot() function, specifying the saved object containing the 2x2 table of counts (e.g., mytable) and other arguments if you want to customize the bar plot (e.g., main= for the plot title, xlab= for the bar labels, and col= for modifying the bar colors):
###Create the stacked barplot
barplot(t(mytable), main="Barplot of Nicotine Gum Usage by Sex",
        xlab="Sex",
        col=c("darkblue","green"))

###Add a legend to the barplot
legend(x=0.25,y=3000,legend=rownames(t(mytable)),
       fill=c("darkblue","green"),
       cex=0.5)

Notice that there is a t() around the table object (mytable). This is to transpose the table so that the groups of interest are placed on the x-axis and the fill is the outcome/response of interest.

For any plot, you can add a legend using the legend() function to describe features of the plot. The arguments that are included in the above code include:

  • the x- and y-coordinates where the legend will be positioned (x= and y=),

  • the text within the legend (legend=) [Notice again that t() was used for the table to be consistent with how the plot was oriented.],

  • the colors that correspond to the text (fill=), and

  • the size (relative to 1) of the legend box (cex=). Values less than 1 reduce the size of the legend and values greater than 1 increase the size of the legend.

Side-by-side bar plot

  • To create a side-by-side bar plot by group, use the barplot() function, specifying the saved object containing the 2x2 table of counts (e.g., mytable), the option for creating side-by-side bars (i.e., beside=TRUE), and other arguments if you want to customize the bar plot (e.g., main= for the plot title, xlab= for the bar labels, and col= for modifying the bar colors):
barplot(t(mytable), beside=TRUE, 
        main="Barplot of Nicotine Gum Usage by Sex",
        xlab="Sex",
        col=c("darkblue","green"))

legend(x=1,y=2500,legend=rownames(t(mytable)),
       fill=c("darkblue","green"),
       cex=0.5)

R TIP

As a reminder, if you want the formal R documentation for a function, type ? before the function name (e.g., ?barplot). This is useful when you want to see what other arguments can be called in the function.

Also, if you curious on the names of the colors that are available in R, see the following website: Colors in R.

10.3.3 Risk difference, relative risk, and odds ratio

When finding RD, RR, and OR in R, you have to first manually create the 2x2 table before using the function that finds the summary values for comparing categorical variables. When creating this table, it is very important you set up the table correctly in order for the function to work. The first column in the table needs to be the event of interest and the first row needs to be the group of interest. As an example, this is how the table should look:

Event Not Event
Group 1 a b
Group 2 c d


  • Step 1 (skip this step if you only have counts and/or totals for each group and not a entire data set): To create and view a 2x2 table of counts, use the table() function, specifying the two variables of interest, exposure or grouping variable first followed by the response or outcome variable next:
table(lhs$AGENDER, lhs$AV1GUM)

Now that you have the counts, you can manually create the 2x2 table. However, notice that the table that was produced by the above code has the event of interest (1) in the second column. Therefore, you cannot just copy the counts down in the order that the table() function displays them. You have to be conscious of order when you manually record the values.

  • Step 2: To create a matrix of a 2x2 table of counts, use the matrix() function to create a table of the data, specifying the counts in the c() function in the order of a, b, c, and d (according the table presented earlier), the number of rows in the table (nrow=2), the number of columns in the table (ncol=2), the order to read in the values (byrow=TRUE fills in the values across the rows), and the row names and column names in the table (dimnames=list(c("Row1Name", "Row2Name"), c("Column1Name", "Column2Name")):
mytable1<-matrix(
    c(387,1708,551,2942), #specifying the cell values
    nrow=2,      #specifying the number of rows
    ncol=2,      #specifying the number of columns
    byrow=TRUE,  #create the matrix by rows
    dimnames=list(c("Female", "Male"),
                  c("Used nicotine gum", "Did not use nicotine gum")))

####View the table that you just created
mytable1

R TIP

It’s good practice when creating 2x2 tables to view the table you just created, especially, in this case, to make sure it was entered in correctly. To do this, you just need to type the name of the object that contains the 2x2 table and run that line of code.


  • Step 3: To find estimates (RD, RR, and OR) for comparing categorical variables, use the summary(epi2x2()) functions, where the epi2x2() function is from the {epibasix} package, specifying the saved object containing the 2x2 table of counts (e.g., mytable1):
#############
# Run the install.packages() function once per package
#############
install.packages("epibasix")

###Access the package using the library() function
library(epibasix)

###Calculate multiple summary statistics using the CrossTable() function
summary(epi2x2(mytable1))

The output presents you everything that you might be interested in knowing when comparing two categorical variables: 2x2 table, hypothesis testing for two inferential methods (Chi-squared test and Fisher’s Exact test; to be presented in a future lesson), and summary measure values for OR, RR, and RD and their confidence intervals.

The confidence intervals for RR and OR are calculated from the log(RR) and the log(OR) and then back-transformed onto the original scale.

10.4.1 Modifying categorical labels

If you examine the variable race in a table, do you notice that the labels are not very intuitive? What does race = 1 represent? How about race = 5? It might be more useful to modify the labels of the categorical variable to something meaningful.

  • To change labels of a categorical variable, first, create a new variable in the data set (e.g., lhs$race1), followed by the assignment operator (<-), and then use the factor() function, specifying the variable of interest (e.g., lhs$race), the current categories of the variable (e.g., levels=c(1, 2, 3, 4, 5)), and the modified categories of the variable in the same order as in the levels= argument (e.g., labels=c("White","Black","Asian","Native American","Other")):
lhs$race1<-factor(lhs$race,
                 levels=c(1,2,3,4,5),
                 labels=c("White","Black","Asian","Native American","Other"))

Now when the new variable, race1, is used, the text labels will be displayed rather than the numeric labels found in race.

Try it out!

mytable2<-table(lhs$race1, lhs$alphagroup)
mytable2

barplot(mytable2, main="Barplot of Race by Intervention Group",
        xlab="Intervention Group",
        col=c("darkblue","green","yellow","orange","red"))

legend(x=2.5,y=1500,legend=rownames(mytable2),
      fill=c("darkblue","green","yellow","orange","red"), 
      cex=0.5)

R TIP

You may have noticed that the c() function is used often in R. This function is used when you have multiple values that you have to specify.

Also, recall that when you want to create a new variable and add it to an existing data set, you just type ( dataset$variablename<-) followed by what you want that variable to contain (e.g., mapping, modifying, transforming, creating).
