Software Lesson 5 Hypothesis Test for a Mean in R

5.1 Goal of Software Lesson 5

The goal of this software lesson is to learn how to

  • obtain test statistics and -values for carrying out a hypothesis test for a mean,
  • obtain inferential results (e.g., CIs and p-values) for a mean difference when the data are paired, and
  • reclassify a variable (continuous or categorical).


Let’s use the Lung Health Study (LHS) data set again.

Recall that the data set and a description of data can be found on Canvas, titled lhs.csv and lhsdoc.docx, respectively.

lhs<-read.csv(file = file.choose(), header = TRUE)

5.2 Hypothesis test for a mean

In a previous software lesson, you learned how to estimate a population mean by computing a confidence interval for that parameter. In that lesson, you estimated the population mean weight change from baseline to final visit after starting treatment.

Now suppose instead of estimating the population mean, you want to test a particular claim about the population mean. The LHS variable that will be the focus in this section is the same variable you obtained an estimate for in the previous software lesson: the variable for weight change (kg) from baseline to 5th annual visit, wgtchg05.

The question that you will be trying to answer here is:

Was there a change in the population mean weight between baseline and the final visit after starting treatment? (i.e., \(\mu = 0\))


  • To carry out a hypothesis test for a mean (when Central Limit Theorem holds), use the t.test() function, specifying the variable of interest, the confidence level (e.g., conf.level=0.95), the appropriate null value (e.g., mu=0), and the appropriate alternative hypothesis (e.g., alternative="two.sided"):
t.test(lhs$wgtchg05, conf.level=0.95, mu=0, alternative="two.sided")

The t.test() function outputs both a hypothesis test for a mean (t-test statistic, degrees of freedom, and p-value) as well as the \(C\%\) confidence interval for a mean, where \(C\%\) is the defined confidence level.

The default/pre-set values for the arguments in the t.test() function are

  • conf.level=0.95,

  • mu=0 (i.e., \(\mu=0\)), and

  • alternative="two.sided".

If those are the values that you need for your test, then you can leave these arguments out of the function.

Try it out!

t.test(lhs$wgtchg05)

However, if you have a different confidence level, null value, or alternative hypothesis direction, then you need to include those additional arguments in the function.

Plausible values that might be used to modify the confidence level include 0.90 (when you want a \(90\%\) CI) or 0.99 (when you want a \(99\%\) CI).

The null value that you input into the mu=value argument depends on the value specified in the null hypothesis for the scenario.

Lastly, the other two options that can be used in the alternative= argument are "less" and "greater", depending on how the alternative hypothesis is specified for the scenario.

HYPOTHESIS TEST FOR A MEAN

Purpose: To evaluate a particular claim about the population mean by using our sample data as evidence.

5.4 Paired t-test

In the LHS study, which was a smoking intervention study, the researchers could be interested in seeing if there was a change in salivary cotinine between the 5th (and last) annual visit (COT5) and baseline (COT0), regardless of the intervention group. This situation can be viewed as a paired data situation because there is a “pre” measurement/variable (baseline) and a “post” measurement/variable (5th visit).

The following would be their research question:

Is the population mean change in salivary cotinine between baseline to 5th annual visit different from 0?


To answer the research question, you would compute a confidence interval to estimate the population mean change and also carry out a hypothesis test for the population mean change. Both of these can be calculated by using the t.test() function (provided that the assumptions for the t-test are valid for the situation).

To carry out a hypothesis test for a mean difference (paired data), there are two options available. In the first option, you would include both variables (pre and post) in the t.test() function but also include an additional argument telling R that the data is paired. In the second option, you would create a new variable of the difference and then use that new variable in the t.test() function.

Option #1

  • To carry out a hypothesis test for a mean difference (paired data), use the t.test() function, specifying the two variables of interest, the confidence level (e.g., conf.level=0.95), the appropriate null value (e.g., mu=0), the appropriate alternative hypothesis (e.g., alternative="two.sided"), and the paired nature of the variables (e.g., paired=TRUE):
t.test(lhs$COT5, lhs$COT0, conf.level=0.95, mu=0, alternative="two.sided", paired=TRUE)

The output for the paired t-test will look very similar to the output when you used t.test() to obtain inferential results for a single mean.

Also notice that all of the usual arguments that you learned previously also apply here (e.g., conf.level=, mu=, alternative=).

Option #2

  • To carry out a hypothesis test for a mean difference (paired data), first, create a new variable in the data set (e.g., lhs$cot5diff), followed by the assignment operator (<-), and then take the difference between the two paired variables. Then, use the t.test() function, specifying the variable of interest, the confidence level (e.g., conf.level=0.95), the appropriate null value (e.g., mu=0), and the appropriate alternative hypothesis (e.g., alternative="two.sided"):
####Creating the change variable
lhs$cot5diff<-lhs$COT5-lhs$COT0

####Carrying out the one-sample t-test on the change variable
t.test(lhs$cot5diff, conf.level=0.95, mu=0, alternative="two.sided")


PAIRED T-TEST

Purpose: To make an inference (CI or hypothesis testing) about the population mean change when there is paired data (e.g., pre/post measurements on an individual).

5.6 Reclassify a variable

There are situations where you might need to reclassify a numerical variable as a categorical one, or recode categories within a categorical variable.

5.6.1 Reclassify numerical variable as categorical variable

Let’s learn how to reclassify a numerical variable from the LHS data set (breathing test value at the 3rd annual visit, FEVFVC3) into a categorical variable. Suppose that you want to turn this variable into a binary variable: normal lung function and abnormal lung function. Based on guidelines, an individual’s lung function is considered “normal” if the ratio between FEV1 and FVC is equal to or greater than \(70\%\) (i.e., \(\frac{FEV_1}{FVC} \ge 70\%\) ). Conversely, an individual’s lung function is considered “abnormal” if the ratio is less than \(70\%\) (i.e., \(\frac{FEV_1}{FVC} < 70\%\)).

  • To reclassify a numerical variable into a categorical one, create a new variable in the data set (e.g., lhs$lung.fxn), followed by the assignment operator (<-), and then use the cut() function, specifying the variable of interest to be converted, the cut-off values in the numerical variable (e.g., breaks=c(0, 69.99999999, 100)), and the labels for the categories (e.g. labels=c("abnormal","normal")):
lhs$lung.fxn<-cut(lhs$FEVFVC3, 
                  breaks=c(0, 69.99999999, 100), 
                  labels=c("abnormal","normal"))

The breaks= argument specifies the cut-offs for how you want to recode the variable. To figure out how many values should be included in the breaks= argument:

  • If you are creating two categories, you would specify three break values.

  • If you are creating three categories, you would specify four break values.

  • etc.

In the FEVFVC3 variable, the minimum value for the ratio is 0 and the maximum is 100. Since you want 70 or greater to be labeled as “normal”, you should set the cut-off mark in the code as a number less than but very close to 70 (e.g., 69.99999999). That way, the abnormal group consists of ratio values between 0 and 69.99999999 and the normal group consists of ratio values between 70 and 100.

R TIP

It’s good practice to view the first few rows of the data to make sure that the software carried out the reclassification as you intended. To refresh you memory on how to view the first few rows, review Software Lesson 1.


5.6.2 Recode categories in a categorical variable

Suppose you want to recode a categorical variable so that “Yes” is labeled as 1 and “No” is labeled as 0. (Note: The 0/1 dummy coding is typically how binary variables are denoted, but it is not always the case.) In the LHS data set, the f060pipe variable, which represents smoking a pipe at the 1st annual visit, is coded not as one would expect: 1=Yes and 2=No. Let’s recode this variable so that 1=Yes and 0=No. 

Option #1: Using mapvalues() function

  • To recode categories within a categorical variable, create a new variable in the data set, followed by the assignment operator (<-), and then use the mapvalues() function from the {plyr} package, specifying the variable of interest to be converted, what the original values were (e.g., from=c(1, 2)), and how to recode the values (e.g., to=c(1, 0)):
#############
# Run the install.packages() function only once per package
#############
install.packages("plyr")

###Access the package using the library() function
library(plyr)

###Recode categories using the mapvalues() function
lhs$pipe1<-mapvalues(lhs$f060pipe, from=c(1, 2), to = c(1, 0))

The order of the values in the from= and to= arguments matter. The first element in from= corresponds to the first element in to=, the second element to the second element, and so on.

R TIP

The c() function has been used a few times in this software lesson. Recall that the c() function is used when there are multiple values that have to be specified.


Option #2: Using ifelse() function

  • To recode categories within a categorical variable, create a new variable in the data set, followed by the assignment operator (<-), and then use the ifelse() function, specifying the variable to be converted and the condition to check, then the new value if the condition is true, and lastly the new value if the condition is false:
###Recode categories as values
lhs$pipe10<-ifelse(lhs$f060pipe == 2, 0, 1)

###Recode categories as text
lhs$pipe10f<-ifelse(lhs$f060pipe == 2, "No", "Yes")

R TIP

As before, it’s good practice to check to make sure the recoding is done correctly. One way to do this when recoding categories of a categorical variable is examine a 2x2 table of the two variables: table(data$new_var, data$old_var).
